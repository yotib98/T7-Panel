## ✨ ویژگی‌های کلیدی

### 🔐 امنیت و دسترسی
- **احراز هویت قوی:** نشست‌های مبتنی بر JWT با کوکی‌های HTTP‑only و امن.
- **مقابله با حملات Brute Force:** محدودیت نرخ برای ورود و تعاملات API.
- **رمزهای عبور سختگیرانه:** سیاست اجباری (حداقل ۸ کاراکتر، حروف بزرگ، کوچک، اعداد).
- **ثبت وقایع (Audit Log):** ثبت تمام تلاش‌های ورود (موفق/ناموفق، IP، مرورگر).

### 📡 مدیریت ورودی‌ها (Inbound)
- **چرخه کامل:** ایجاد، ویرایش، فعال/غیرفعال‌سازی و حذف ایمن کانفیگ‌های VLESS.
- **کنترل دقیق:** محدودیت ترافیک به ازای هر کاربر (گیگابایت)، روزهای انقضا و حداکثر اتصالات همزمان.
- **مسیریابی پیشرفته:** مسیر سفارشی، SNI، Host و اثر انگشت TLS برای هر ورودی.
- **عملیات گروهی:** فعال‌سازی، غیرفعال‌سازی، بازنشانی یا حذف گروهی کانفیگ‌ها.
- **هسته تغییرناپذیر:** ورودی پیش‌فرض `SulgX` به‌طور سیستماتیک در برابر حذف تصادفی محافظت می‌شود.

### 📊 تحلیل لحظه‌ای
- **موتور سرعت زنده:** نمودارهای بسیار دقیق دانلود/آپلود با فیلترینگ تطبیقی پیک‌ها.
- **معیارهای پویا:** میله‌های ترافیک ۲۴ ساعته (آگاه از منطقه زمانی) و نمودارهای دونات توزیع.
- **سلامت سیستم:** نظارت زنده بر CPU، حافظه و دیسک با بازگشت به `loadavg`.

### 🗺️ آی‌پی پاک و اسکنر امن
- **مدیریت آی‌پی:** افزودن، ویرایش و وارد کردن انبوه آی‌پی‌های IPv4/IPv6 که به صورت پویا به اشتراک‌ها متصل می‌شوند.
- **اسکنر امن:** اسکن پورت ۴۴۳ در ۲۴ ارائه‌دهنده ابری از پیش تعریف‌شده (Cloudflare، AWS، Azure و غیره).
- **مقابله با خرابی:** مدیریت ایمن محدوده‌های CIDR عظیم (مثلاً `/14`) با محدود کردن به ۴,۰۹۶ آی‌پی برای جلوگیری از یخ‌زدگی مرورگر. به‌طور خودکار DNSهای عمومی (8.8.8.8) را حذف می‌کند.

### 🤖 ربات تلگرام هوشمند
- **دو زبانه (EN/FA):** قالب‌های کاملاً قابل ترجمه.
- **هشدارهای رویداد:** ورود به پنل، کاربران منقضی‌شده، خطاها و هشدار ۹۰٪ مصرف سهمیه.
- **پیش‌نمایش زنده:** رندر قالب JSON به‌صورت لحظه‌ای در داشبورد.

## 🚀 شروع سریع و استقرار

> [!NOTE]
> SulgX روی *هر* پلتفرمی که از برنامه‌های ASGI پایتون (Uvicorn/Gunicorn) و اتصالات WebSocket استاندارد پشتیبانی کند، اجرا خواهد شد.

### 🍴 مرحله ۱: فورک مخزن
1. به مخزن اصلی در گیت‌هاب بروید: [https://github.com/SulgX/SulgX-Panel](https://github.com/SulgX/SulgX-Panel)
2. روی دکمه **Fork** در بالای صفحه کلیک کنید.
3. در پنجره باز شده، حساب کاربری خود را به عنوان مقصد انتخاب کنید و منتظر بمانید تا فورک کامل شود.
4. اکنون یک کپی کامل از پروژه زیر حساب گیت‌هاب خود دارید که می‌توانید آن را دلخواه تغییر دهید.

### ☁️ مرحله ۲: ثبت‌نام در یک پلتفرم ابری
یکی از سه پلتفرم توصیه‌شده زیر را انتخاب کرده و ثبت‌نام کنید (معمولاً می‌توانید مستقیماً با حساب گیت‌هاب خود وارد شده و دسترسی بدهید):

- [**Render**](https://render.com/) ← توصیه اصلی، بدون نیاز به کارت اعتباری
- [**Railway**](https://railway.app/) ← رابط مدرن، اعتبار اولیه رایگان
- [**Dockfly**](https://dockfly.app/) ← ساده و کم‌حجم

### 🚀 مرحله ۳: استقرار پروژه

<details>
<summary><b>🔹 استقرار روی Render</b></summary>

1. در داشبورد Render، روی **New +** کلیک کرده و **Web Service** را انتخاب کنید.
2. در بخش اتصال گیت‌هاب، مخزن فورک‌شده خود (`SulgX-Panel`) را پیدا کرده و روی **Connect** کلیک کنید.
3. Render به‌طور خودکار فایل `render.yaml` را می‌خواند. نام سرویس و شاخه را تأیید کنید.
4. به پایین بروید تا بخش **Environment Variables** را ببینید و متغیرهای زیر را وارد کنید:
   - `ADMIN_PASSWORD`
   - `SECRET_KEY`
   - `DOMAIN`  
   *(مقدارها باید مطابق جدول متغیرهای محیطی در ادامه همین بخش باشد.)*
5. روی **Create Web Service** کلیک کنید. پس از چند دقیقه، آدرس عمومی سرویس شما تولید می‌شود (مثلاً `sulgx-test.onrender.com`).

</details>

<details>
<summary><b>🔹 استقرار روی Railway</b></summary>

1. در Railway، روی **New Project** کلیک کرده و **Deploy from GitHub repo** را انتخاب کنید.
2. مخزن فورک‌شده خود را انتخاب کنید.
3. Railway به‌طور خودکار `Procfile` را تشخیص می‌دهد. برای افزودن متغیرهای محیطی، به برگه **Variables** رفته و `ADMIN_PASSWORD`، `SECRET_KEY` و `DOMAIN` را با مقادیر مناسب اضافه کنید.
4. ساخت خودکار شروع شده و یک دامنه عمومی دریافت خواهید کرد.

> [!TIP]
> 🗝️ کاربران Railway: محدوده آی‌پی‌های Railway را به لیست آی‌پی پاک اضافه کنید تا اسکن دقیق‌تری داشته باشید.

</details>

<details>
<summary><b>🔹 استقرار روی Dockfly</b></summary>

1. در Dockfly، یک **New Project** بسازید، منبع را **GitHub** قرار دهید و مخزن فورک‌شده خود را انتخاب کنید.
2. در بخش Environment، همان سه متغیر را اضافه کنید.
3. اگر دستور شروع به‌طور خودکار اجرا نشد، دستور زیر را به‌صورت دستی وارد کنید:

```bash
gunicorn -k uvicorn.workers.UvicornWorker main:app --bind 0.0.0.0:$PORT
```

4. روی **Deploy** کلیک کنید.

</details>

### 📌 متغیرهای محیطی
باید متغیرهای محیطی زیر را در داشبورد ارائه‌دهنده خود تنظیم کنید:

| متغیر | مقدار نمونه | توضیح |
| :--- | :--- | :--- |
| `ADMIN_PASSWORD` | `StrongPass!123` | برای دسترسی به پنل لازم است (حداقل ۸ کاراکتر، حروف بزرگ و کوچک، اعداد). |
| `SECRET_KEY` | `random_long_string` | برای امنیت کوکی‌های ورود JWT استفاده می‌شود. |
| `DOMAIN` | `sulgx.up.railway.app` | دامنه عمومی شما. *برای تولید صحیح لینک‌ها بسیار توصیه می‌شود.* |
| `DB_PATH` | `/tmp/panel.db` | محل ذخیره پایگاه داده SQLite. در صورت استفاده از حجم‌های پایدار از `/data/panel.db` استفاده کنید. |

### 📌 دستور شروع
این دستور را دقیقاً روی تمام پلتفرم‌ها استفاده کنید (در صورت نیاز به وارد کردن دستی):

```bash
gunicorn -k uvicorn.workers.UvicornWorker main:app --bind 0.0.0.0:$PORT
```

## ☁️ پلتفرم‌های استقرار

SulgX به گونه‌ای ساخته شده است که روی ارائه‌دهندگان PaaS ابری به‌خوبی کار کند. بدون نیاز به Dockerfile یا تنظیمات پیچیده.

### 🏆 بهترین ارائه‌دهندگان توصیه‌شده

| پلتفرم | محدودیت لایه رایگان | WebSocket | حالت خواب | نیاز به کارت اعتباری؟ | روش استقرار |
| :--- | :--- | :---: | :---: | :---: | :--- |
| **Render** | ۷۵۰ ساعت / ماه | ✅ | بله (با تأخیر) | خیر | خودکار از طریق `render.yaml` |
| **Railway** | ۵ دلار اعتبار اولیه | ✅ | خیر | خیر | خودکار از طریق `Procfile` |
| **Dockfly** | ۱ پروژه (۲۵۶ مگابایت) | ✅ | خیر | خیر | دستور شروع دستی |

## 📁 معماری مخزن

مخزن به‌عمد حداقلی نگه داشته شده است. همه چیز مورد نیاز برای تولید در آن گنجانده شده است:

| فایل | نوع | هدف |
| :--- | :---: | :--- |
| `main.py` | **هسته** | قلب تپنده SulgX. شامل بک‌اند FastAPI، تونل‌های WebSocket و فرانت‌اند HTML/JS توکار. |
| `requirements.txt` | **پیکربندی** | وابستگی‌های پایتون با نسخه‌های دقیق برای تضمین پایداری ساخت. |
| `Procfile` | **استقرار** | دستورالعمل‌های شروع استاندارد برای Heroku، Railway و Render. |
| `render.yaml` | **استقرار** | نقشه زیرساخت به‌عنوان کد برای استقرار یک‌کلیکی روی Render. |
| `sulgx-config.toml` | **مستندات** | راهنمای مرجع حاوی متغیرهای محیطی مورد نیاز برای تنظیمات دستی. |
| `.gitignore` | **Git** | مخزن را با حذف لاگ‌ها، کش‌ها و فایل‌های محلی `.db` تمیز نگه می‌دارد. |

## 💸 راهنمای پهنای باند و قیمت‌گذاری

> [!IMPORTANT]
> **پنل SulgX ۱۰۰٪ رایگان است.** اما ارائه‌دهنده ابری شما بابت پهنای باندی که کاربرانتان مصرف می‌کنند، از شما هزینه دریافت می‌کند.

| پلتفرم میزبانی | پهنای باند رایگان شامل‌شده | هزینه هر گیگابایت اضافی (تقریبی) |
| :--- | :--- | :--- |
| **Render** | ۵ گیگابایت / ماه | `۰.۱۰ دلار / گیگابایت` |
| **Railway** | پرداخت به میزان مصرف | `۰.۱۰ دلار / گیگابایت` |
| **Koyeb** | ۵ گیگابایت / ماه | `۰.۰۴` تا `۰.۱۰ دلار / گیگابایت` |
| **Fly.io** | بسته به منطقه متغیر است | `۰.۰۲ دلار / گیگابایت` |
| **Oracle Cloud** | ۱۰ ترابایت / ماه | نرخ استاندارد ابری |

*داشبورد صورتحساب ارائه‌دهنده ابری خود را برای جلوگیری از هزینه‌های غیرمنتظره زیر نظر داشته باشید. برای کنترل مصرف از محدودیت‌های ماهانه پنل استفاده کنید.*

## ⚖️ سلب مسئولیت سخت‌گیرانه

> [!WARNING]
> **قبل از استقرار با دقت بخوانید**

* **رایگان و غیرتجاری:** این نرم‌افزار کاملاً رایگان ارائه می‌شود. **برای فروش نیست.**
* **ممنوعیت VPNهای تجاری:** از این پنل برای فروش اشتراک VPN استفاده نکنید. این پنل صرفاً برای اهداف شخصی، آموزشی و آزمایشی طراحی شده است.
* **ممنوعیت سوءاستفاده از پلتفرم:** از لایه‌های رایگان ارائه‌دهندگان ابری با ایجاد چندین حساب با ایمیل‌های موقت سوءاستفاده نکنید.
* **گزارش:** اگر مشاهده کردید که کسی دسترسی به این پنل خاص را می‌فروشد یا از زیرساخت سوءاستفاده می‌کند، لطفاً به ارائه‌دهنده میزبانی مربوطه گزارش دهید.
* **عدم پذیرش مسئولیت:** توسعه‌دهنده هیچ‌گونه مسئولیتی در قبال خسارات، هزینه‌های اضافی یا نقض شرایط استفاده نمی‌پذیرد. شما به‌تنهایی مسئول ترافیک خود هستید.
